# ML Final Project Criteria

_Source file: ML Final Project Criteria.pdf_

## Deliverables

### Machine Learning Model & Source Code

- Link to the model and source code.
- Upload in Github.

### Deployed Application

- Must be deployed as a usable application.
- Web-based or API-based.
- Must support: input -> execute -> output.
- Accepted demo options: local demo, shared access link, or recorded live demo.

### Report

- Format: PPT or PPTX.
- Length: 15-25 slides, excluding cover and appendix.
- Language: English or Indonesian.
- Slides must be self-explanatory.

### Demo Video

- Maximum duration: 10 minutes.

## Required PPT Structure

1. Cover
2. Problem Statement & Motivation
3. Dataset Description
4. Exploratory Data Analysis (EDA)
5. ML Problem Formulation
6. Model Selection & Rationale
7. Training & Validation Setup
8. Evaluation Metrics & Results
9. Deployment Architecture
10. Application Screenshots
11. User Testing Design
12. User Testing Results
13. Analysis & Discussion
14. Limitations
15. Conclusion & Future Work
16. Appendix (optional)

## Real User Testing Requirements

Mandatory requirements:

- Minimum 5 users.
- Students must define:
  - anonymized user background;
  - usage scenario;
  - usability and usefulness feedback.
- Collect quantitative feedback, such as rating or Likert scale.
- Collect qualitative feedback.
- Results must be analyzed and presented in the PPT.

## Technical Notes

- Programming language: Python.
- Libraries: NumPy, Pandas, Scikit-learn, Matplotlib, Seaborn.
- Deployment frameworks, mandatory: Flask, FastAPI, or Streamlit.
- Version control: Git, mandatory.

## Project Proposal Checkpoint

The proposal should include:

- Project title and problem statement.
- Dataset source and description.
- Selected ML methods and baseline.
- Planned evaluation metrics.
- Deployment and user testing plan.
- References, minimum 2-3.
- Week 6 submission: 1-2 pages, PDF.
